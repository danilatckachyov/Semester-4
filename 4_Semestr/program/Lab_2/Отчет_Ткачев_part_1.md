## 2. Примеры запросов и ответов API

### Создание книги (POST /api/v1/books)
**Запрос:**
```json
{
  "title": "Преступление и наказание",
  "author": "Фёдор Достоевский",
  "genre": "fiction",
  "publication_year": 1866,
  "pages": 671,
  "isbn": "9781234567898"
}
```
**Ответ:**
```json
{
  "id": 1,
  "title": "Преступление и наказание",
  "author": "Фёдор Достоевский",
  "genre": "fiction",
  "publication_year": 1866,
  "pages": 671,
  "isbn": "9781234567898",
  "available": true
}
```

### Получение всех книг (GET /api/v1/books)
**Ответ:**
```json
[
  {
    "id": 1,
    "title": "Преступление и наказание",
    "author": "Фёдор Достоевский",
    "genre": "fiction",
    "publication_year": 1866,
    "pages": 671,
    "isbn": "9781234567898",
    "available": true
  }
]
```

### Заимствование книги (POST /api/v1/books/1/borrow)
**Запрос:**
```json
{
  "borrower_name": "Иван Иванов",
  "return_days": 14
}
```
**Ответ:**
```json
{
  "id": 1,
  "title": "Преступление и наказание",
  "author": "Фёдор Достоевский",
  "genre": "fiction",
  "publication_year": 1866,
  "pages": 671,
  "isbn": "9781234567898",
  "available": false,
  "borrowed_by": "Иван Иванов",
  "borrowed_date": "2026-03-26",
  "return_date": "2026-04-09"
}
```

### Ошибка при попытке удалить взятую книгу (DELETE /api/v1/books/1)
**Ответ:**
```json
{
  "detail": "Нельзя удалить взятую книгу"
}
```

## 3. Ответы на вопросы

### 1. В чем преимущества использования Pydantic моделей для валидации?
- Pydantic позволяет описывать структуру и ограничения данных декларативно через Python-классы.
- Валидация происходит автоматически при получении запроса, ошибки возвращаются в удобном формате.
- Поддерживается автогенерация примеров, документации и строгая типизация.
- Упрощает поддержку и расширение API, снижает вероятность ошибок.

### 2. Как работает автоматическая документация в FastAPI?
- FastAPI автоматически генерирует документацию Swagger UI и ReDoc на основе описанных моделей, типов и аннотаций.
- Документация доступна по адресам `/docs` и `/redoc`.
- Все параметры, типы, ограничения и примеры отображаются в UI без дополнительного кода.

### 3. Почему важно проверять уникальность ISBN?
- ISBN — уникальный идентификатор книги. Если не проверять уникальность, в каталоге могут появиться дубликаты.
- Это приведет к путанице при поиске, заимствовании и возврате книг.
- Проверка уникальности — важное требование для корректной работы библиотеки.

### 4. Какие статус-коды HTTP вы использовали и почему?
- **201 Created** — при успешном создании книги (POST)
- **200 OK** — при успешных операциях получения, обновления, возврата
- **204 No Content** — при успешном удалении книги
- **400 Bad Request** — при ошибках валидации или бизнес-логики (например, попытка удалить взятую книгу)
- **404 Not Found** — если книга не найдена по ID
