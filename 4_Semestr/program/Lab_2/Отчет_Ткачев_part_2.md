## 2. Примеры запросов и ответов API

### Создание задачи (POST /api/tasks)
**Запрос:**
```json
{
  "title": "Купить продукты",
  "description": "Молоко, хлеб, яйца",
  "category": "shopping",
  "priority": 2,
  "dueDate": "2024-12-31"

```
**Ответ:**
```json
{

  "message": "Задача успешно создана",
  "data": {
    "id": 1,
    "uuid": "...uuid...",
    "title": "Купить продукты",
    "description": "Молоко, хлеб, яйца",
    "category": "shopping",
    "priority": 2,
    "dueDate": "2024-12-31",
    "completed": false,
    "createdAt": "2026-03-26T12:00:00.000Z",

  }
}
```

## 3. Ответы на вопросы

### 1. Какие middleware вы использовали и для чего?
- **helmet** — для базовой защиты HTTP-заголовков.
- **cors** — для разрешения кросс-доменных запросов.
- **express-rate-limit** — для ограничения частоты запросов.
- **express.json()** — для парсинга JSON-тел запросов.
- **custom validation** — для валидации данных задач (Joi).
- **errorHandler** — для централизованной обработки ошибок.
- **notFoundHandler** — для обработки 404.

### 2. Как работает валидация с Joi в сравнении с Pydantic из части 1?
- Joi — библиотека для валидации данных в Node.js, позволяет гибко описывать схемы и сообщения об ошибках.
- Pydantic (Python) — аналогично, но интегрирован с типизацией Python и FastAPI, поддерживает автогенерацию OpenAPI.
- Оба инструмента позволяют централизованно валидировать входные данные и возвращать подробные ошибки.

### 3. В чем преимущества файлового хранения данных для этого задания?
- Простота реализации, не требуется отдельная СУБД.
- Данные легко просматривать и редактировать вручную.
- Подходит для учебных и небольших проектов.
- Недостатки: не подходит для больших нагрузок и параллельных изменений.

### 4. Как бы вы улучшили это API для production использования?
- Перейти на полноценную СУБД (PostgreSQL, MongoDB и др.).
- Добавить аутентификацию и авторизацию пользователей.
- Реализовать логирование в файл и мониторинг ошибок.
- Добавить автоматические тесты и CI/CD.
- Использовать переменные окружения для конфигурации.
- Ограничить доступ к API (CORS, rate limit, security best practices).
